## Overview
Access key persistence review checks whether cloud identities have stale, newly created, or over-privileged long-lived credentials.

## Steps
- Confirm identity review is in scope.
- List access keys and creation dates.
- Check last-used service and region.
- Correlate key creation with test activity.
- Recommend rotation, removal, or short-lived credentials.

## Evidence
Record principal, key id, creation date, last used time, service, and remediation action.
