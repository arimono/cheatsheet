## Overview
Rate limit testing checks whether an API can resist credential attacks, OTP abuse, scraping, and expensive request floods.

## Steps
- Confirm limits and test volume in the rules of engagement.
- Measure baseline behavior before increasing request count.
- Test per-account, per-IP, per-token, and per-endpoint controls.
- Watch for lockouts, CAPTCHA, throttling headers, and delayed responses.

## Evidence
Record request count, time window, response pattern, and business impact.
